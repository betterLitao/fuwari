## 核心问题

相比单轮 LLM 对话，智能体由于递归调用内存、工具和规划，导致了指数级的资源消耗——上下文窗口饱和、推理步骤过多、token 成本失控。

单纯的模型压缩不足以解决问题，必须从智能体系统本身的**记忆管理、工具调用和规划过程**入手进行优化。

## 效率的定义

在固定的成本预算（Token 数、延迟、计算量）下最大化任务成功率，或在相同效果下最小化成本。

---

## 一、高效记忆

### 记忆构建

**工作记忆（直接参与推理的上下文）：**
- **文本记忆**：通过大模型提取关键信息并压缩成简短摘要，避免 Lost in the Middle
- **潜在记忆**：将长上下文压缩成 KV 缓存或隐藏状态，以极低计算代价保存信息

**外部记忆（模型外存储）：**
- **基于项目的记忆**：提取和压缩对话记录来减少 Token 消耗
- **基于图的记忆**：构建知识图谱，组织实体关系支持多跳检索
- **层次化记忆**：OS 式层级结构（FIFO 缓存、段落摘要）管理海量信息

### 记忆管理

- **规则驱动**：艾宾浩斯遗忘曲线衰减、FIFO 淘汰。低成本，但可能丢失关键信息
- **LLM 驱动**：由 LLM 决定 ADD/DELETE 操作。自适应，但增加计算开销
- **混合驱动**：规则触发阈值 + LLM 语义合并，必要时才调用 LLM 压缩

### 多智能体记忆

- **共享记忆**：构建多层图结构，避免冗余上下文
- **本地记忆**：固定大小模块存储路由信息，减少跨智能体检索成本

## 二、高效工具学习

### 工具选择

面对海量工具库，直接在 Prompt 中塞入所有工具描述不可行：

- **外部检索器**：对比学习将查询和工具描述嵌入语义空间，相似度检索
- **多标签分类**：训练小模型直接预测需要的工具集合
- **词汇检索**：将工具嵌入词表，模型生成特定 Token 时即触发工具调用

### 工具调用

- **原位填充**：LLM 在生成过程中直接填充工具参数
- **并行调用**：分析工具调用图，无依赖的工具并行执行，降低延迟
- **成本感知**：将工具调用视为背包问题，在预算约束下选择最优工具组合

---

## 三、高效规划

### 单智能体规划

效率问题主要来源于"想太深、搜太贵"：

- **选择性思考**：快思考/慢思考机制，根据任务复杂度动态分配推理预算
- **结构化搜索**：A\* 搜索式启发评估函数，剪去不必要的搜索分支
- **任务分解**：先规划后执行，将大问题分解为多个小问题
- **记忆与技能复用**：复用已存储的经验，越用越省

### 多智能体协作

效率问题主要来源于"聊太多、通信太重"：

- **拓扑效率**：稀疏压缩的通信拓扑降低交互成本
- **协议优化**：精简上下文信息，确保每次通信携带高价值信息
- **协同蒸馏**：教师-学生式知识迁移，跨智能体共享策略

## 评估方法

传统的准确率指标无法全面衡量成本，建议采用**成本-有效性曲线（Pareto Front）**来评估。

常见效率指标：
- Token 消耗与 API 成本
- 端到端延迟、推理时间
- GPU 内存使用量
- 每次响应的平均 LLM 调用次数

---

## 未来方向

1. **统一的效率评估框架**：当前不同方法使用不同的效率维度和术语，难以直接比较
2. **智能体潜在推理**：在连续隐藏表示中计算，减少 token 开销并保留更丰富的信息
3. **面向部署的设计**：多模型部署 vs 单模型角色扮演流水线，在匹配资源预算下比较方案
4. **多模态智能体效率**：长程多模态任务的视觉历史维护，累积计算负担更严重
